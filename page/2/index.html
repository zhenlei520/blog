  4. 机密管理器（仅Windows）
  5. 环境变量配置提供程序（未限定前缀）
  6. 命令行配置提供程序

异或

异或是什么？

异或（ xor）是一个 数学运算符

它应用于 逻辑运算。异或符号为 ^

异或的3个特点：

  1. 0^0=0,0^1=1 0异或任何数＝任何数

  2. 1^0=1,1^1=0 1异或任何数－任何数取反

  3. 1^1=0,0^0=0 任何数异或自己＝把自己置0

docker安装redis

1. 搜索查找镜像

docker search redis

2. 下载镜像

docker pull redis:latest

3. 查看镜像并创建redis和redis/data目录

docker images

4. 使用镜像创建容器

docker run -p 6379:6379 -v $PWD/data:/data -d redis:latest redis-server --appendonly yes

windows： docker run -p 6379:6379 -v /E:/ProgramFiles/Docker-Redis:/data -d redis:latest redis-server --appendonly yes 

-p   6379:6379 :将容器的6379端口映射到主机的6379端口
-v   $PWD/data:/data :将主机中当前目录下的data挂载到容器的/data
redis-server --appendonly yes  在容器执行redis-server启动命令，并打开redis持久化配置

5. 进入容器操作redis

docker run -it redis:latest redis-cli -h 192.168.163.170或使用docker run -it 标识

endpoints

错误信息：

EndpointRoutingMiddleware matches endpoints setup by EndpointMiddleware and so must be added to the request execution pipeline before EndpointMiddleware. Please add EndpointRoutingMiddleware by calling ‘IApplicationBuilder.UseRouting’ inside the call to ‘Configure(…)’ in the application startup code.

解决办法：

1. 在使用UseEndpoints之前增加UseRouting

IEndpointRouteBuilder app;//假使此时app是有值的
app.UseRouting();
app.UseEndpoints(endpoint =>
{
    endpoint.Map("/api/v1/payment/HelloWorld", HelloWorld);
});

2. 或者直接使用

app.Map("/api/v1/payment/HelloWorld", HelloWorld);

UseRouting与UseEndpoints区别：

UseRouting：将EndpointRoutingMiddleware中间件注册到http管道中
UseEndpoints：将本程序集定义的EndPoint放到路由中间件的配置对象RouteOptions中
将EndpointMiddleware中间件注册到http管道中

UseRouing 中间件主要是路由匹配，找到匹配的终结者路由Endpoint ；UseEndpoints 中间件主要针对UseRouting 中间件匹配到的路由进行 委托方法的执行等操作。

Aop

Aop的织入原理分为动态织入以及静态织入

动态织入：

程序运行过程中织入的是动态织入，因此也被称为运行时增强（基于接口实现、基于类实现）。在.NET中可以有两种实现：采用“装饰者模式”设计项目类库来实现；基于透明代理(TransparentProxy)/真实代理(RealProxy)来实现

静态织入：

通过扩展编译器对代码的中间语言(IL)插入代码的方式实现对目标对象的调用拦截，在编译时期就织入，即编译出来的class文件，字节码就已经被织入了，从而在编译阶段就可生成Aop代理类，因此也被称为编译时增强（特殊编译器实现、特殊的类加载器实现）

一、Dapr安装

Windows

步骤1：使用管理员打开Powershell

Set-ExecutionPolicy RemoteSigned -scope CurrentUser

步骤2：执行安装命令

iwr -useb https://raw.githubusercontent.com/dapr/cli/master/install/install.ps1 | iex

步骤3：验证dapr是否安装成功

dapr

步骤4：验证Dapr版本

dapr --version 或者 dapr -v

步骤5：运行init cli命令

dapr init

步骤6：验证容器正在运行

docker ps

请确保镜像为daprio/dapr, openzipkin/zipkin和 redis 的容器都在运行：

疑问：

  1. 执行步骤2出错，挂代理安装再试（需要使用powershell运行安装命令）

  2. dapr出错，使用管理员权限重新打开powershell、再运行dapr即可

  3. 当运行dapr init出错，提示以下错误：

    Making the jump to hyperspace…
    Failed to get runtime version: ‘https://api.github.com/repos/dapr/dapr/releases - 403 Forbidden’. Trying secondary source
    cannot get the latest release version: ‘Get “https://dapr.github.io/helm-charts/index.yaml": read tcp 192.120.5.172:55698->185.199.111.153:443: wsarecv: An existing connection was forcibly closed by the remote host.’. Try specifying –runtime-version=

    解决办法：根据当前安装的dapr版本，初始化时选择对应版本安装，例如：dapr init –runtime-version=1.3.0 (需要挂全局代理安装)

dotnet_nuget

nuget 打包

dotnet pack –configuration Release -p:IncludeSymbols=true -p:SymbolPackageFormat=snupkg –output nupkgs
或者简写为
dotnet pack -c Release -p:IncludeSymbols=true -p:SymbolPackageFormat=snupkg –output nupkgs

–output：输出路径
–configuration：环境配置
-p:IncludeSymbols 包含symbols
-p:SymbolPackageFormat 输出符号包 (.snupkg)

查看文档

nuget 推送

dotnet nuget push *.nupkg –key %key% –source https://nuget.pkg.github.com/zhenlei520/index.json –skip-duplicate
或者简写为
dotnet nuget push *.nupkg -k %key% -s https://nuget.pkg.github.com/zhenlei520/index.json –skip-duplicate

–source：nuget package source
–key：nuget api key、用于管理nuget包
–skip-duplicate：跳过错误继续推送

查看文档

nuget 删除

dotnet nuget delete package version -s https://nuget.pkg.github.com/zhenlei520/index.json -key 秘钥

例如：
dotnet nuget delete Microsoft.AspNetCore.Mvc 1.0 -s https://nuget.pkg.github.com/zhenlei520/index.json -key 秘钥

三、静态类与单例

两者区别

  1. 单例可以继承类以及被继承和实现接口以及支持重写（override），而静态类不能继承和被继承以及实现接口以及重写（override）
  2. 静态类是在首次加载时初始化，而单例是在其静态对象被使用时加载初始化（延迟初始化）
  3. 单例对象存储在堆中，而静态类存储在堆栈中
  4. 单例对象有构造函数、静态类则没有
  5. 单例对象可以克隆、但静态类不能被克隆
  6. 静态类比单例模式的效率更高，因为其在编译期就完成了静态绑定
  7. 单例是受控制的静态对象，如果类对象被释放了、单例对象也会被释放，而静态类则不会被释放，仅在项目退出才释放

什么情况下使用静态类

  1. 类中没有任何状态性质的属性，只是提供了一些公共的方法

什么情况下使用单例

  1. 类中需要维护一些状态信息
  2. 需要被克隆

单例优点

可以通过继承实现多态、且可以被克隆、在类对象被释放后也会随之释放
